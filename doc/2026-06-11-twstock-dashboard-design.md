# 台股分析儀表板 設計規格
日期：2026-06-11　狀態：使用者已口頭核准設計，待書面確認

## 目的
盤後執行一次，產出本機網頁儀表板，回答三個問題：
1. 重要國家大盤今天表現如何？
2. 今天的資金集中在哪些產業族群？
3. 哪些族群正在醞釀突破（下一階段主流）？

## 使用方式
- 收盤後（建議 15:00 之後，待證交所資料更新）執行 `python run.py`
- 雙擊 `web/dashboard.html` 於瀏覽器檢視
- 無伺服器、無資料庫；歷史資料以 JSON 檔沉澱於 `data/history/`

## 架構
```
twstock-dashboard/
├── run.py            # 進入點：fetch → score → render data.js
├── fetchers.py       # 所有外部資料抓取
├── scoring.py        # 族群熱度與突破口評分（純函式，可測試）
├── data/history/YYYY-MM-DD.json   # 每日快照
└── web/
    ├── dashboard.html   # 單檔前端（內嵌 CSS/JS，Chart.js CDN）
    └── data.js          # run.py 產出，window.DASHBOARD_DATA = {...}
```
模組邊界：fetchers 只負責取得原始資料並正規化為 DataFrame；scoring 只吃正規化資料、回傳評分結構；run.py 串接與輸出。scoring 不得進行任何網路存取。

## 資料來源（免費、免金鑰）
| 用途 | 來源 |
|---|---|
| 國際指數（^TWII, ^GSPC, ^IXIC, ^SOX, ^N225, ^KS11, 000001.SS, ^HSI）近60日 | yfinance |
| 上市個股日成交 | TWSE OpenAPI `exchangeReport/STOCK_DAY_ALL` |
| 上市公司產業別 | TWSE OpenAPI `opendata/t187ap03_L` |
| 上櫃個股日成交與產業別 | TPEx OpenAPI 對應端點 |
| 三大法人個股買賣超（上市/上櫃） | TWSE `fund/T86`、TPEx 對應端點 |

## 功能規格

### F1 國際大盤
- 八指數卡片：收盤、漲跌%、5日%、20日%
- 折線圖：近60日標準化走勢（基期=100），台股 vs 費半 vs S&P500 可勾選
- 台股–費半 20 日相關係數顯示

### F2 當下熱門族群
- 依產業別（證交所分類，上市櫃合併）彙總：
  - 當日類股平均漲跌%（成交金額加權）
  - 類股成交金額占全市場比重
  - 漲停家數、創20日新高家數
- 輸出熱度榜 Top 10 表格＋全族群成交占比橫條圖

### F3 突破口綜合評分（0–100）
- 量能 40 分：類股成交金額占比之5日斜率（20）＋創20日新高家數之3日變化（20）
- 籌碼 30 分：三大法人近3日買超金額／類股總市值之強度（15）＋法人連續買超天數（15）
- 輪動 30 分：20日漲幅位於全族群後 50%（低基期，15）＋5日相對大盤強度由負轉正（15）
- 各子項以全族群分位數正規化後加權
- 儀表板顯示每族群評分＋近3日評分變化箭頭（↑連升才視為有效訊號）
- 評分明細可展開：顯示該族群法人買超前5名個股

### 歷史快照
- 每日將原始彙總與評分寫入 `data/history/YYYY-MM-DD.json`
- 3日變化箭頭由歷史檔計算；不足3日時顯示「資料累積中」

## 錯誤處理
- 休市判定：STOCK_DAY_ALL 回傳空或日期非今日 → 顯示「今日休市/資料未更新」並退出
- 各 API 重試3次（間隔2秒）；單源失敗時沿用最近一份歷史快照之該部分，頁面以黃色標籤標示「資料延遲：來源名」
- yfinance 個別指數失敗不阻斷其餘指數

## 測試
- scoring.py 附 pytest：以固定假資料驗證（a）滿分/零分邊界（b）分位數正規化（c）3日箭頭邏輯
- fetchers 提供 `--mock` 模式以本地樣本 JSON 跑通全流程
